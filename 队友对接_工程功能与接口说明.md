# 小车工程功能、接口与队友对接说明

最后更新：2026-07-23
当前基线版本：V8 / Git 提交 `432241d`
工程根目录：`F:\car\code`

## 1. 对接前先看这里

1. CCS 只打开 `F:\car\code`，不要再创建或导入第二套嵌套工程。
2. 唯一 SysConfig 文件是 `F:\car\code\empty.syscfg`。
3. `Debug` 是生成目录，可以 Clean 后重建，不要把里面的
   `ti_msp_dl_config.c/.h` 当作手写源码修改。
4. 当前负责的直线段和第三题代码已经模块化：
   - 第一、二题公共调度：`mission.c/.h`
   - 第三题独立状态机：`q3_mission.c/.h`
   - 队友写循迹时应通过预留接口衔接，不要直接修改第三题内部静态状态。
5. 当前 PB24/第四题只完成按键枚举和引脚配置，没有实现任务状态机。
6. 所有按钮均为上拉输入、按下低电平。按钮绝对不能配置为 GPIO 输出。

## 2. 当前已经实现的功能

### 2.1 上电与安全

- 上电后先拉低 TB6612 的 STBY 和四个方向引脚，再启动 PWM。
- 空闲、完成和故障状态持续执行 `Motor_stop()`。
- PWM 初始比较值为 0。
- 每个按键必须先独立检测到松开高电平 100 ms 才会解锁。
- 每个按键独立消抖 20 ms；一路按键故障不会锁住其他按键。
- LCD 显示以开机首次有效陀螺仪角度为 0°的相对角度。

### 2.2 第一题：A→B

启动按钮：A18。

当前流程：

1. 按 A18，发出起点声光提示。
2. 编码器清零。
3. 小车以目标 0°直行：
   - 陀螺仪有效时使用 PID 保持 0°；
   - 陀螺仪无效时退化为左右轮等速。
4. 目标距离为 1000 mm。
5. 行驶到约 750 mm 后，从 360‰ 降到 160‰。
6. 满足下列任意条件即进入停车：
   - 至少两路灰度连续 20 ms 检测到黑线；
   - 编码器距离达到 1000 mm。
7. 以 280‰ 反向驱动 100 ms，随后关闭电机，发出完成声光提示。

黑线是正常终点条件，编码器 1000 mm 是没有黑线时的硬距离兜底。

### 2.3 第二题：A→B、C→D

启动按钮：PB22。

当前 A→B 流程：

1. 按 PB22，发出起点声光提示。
2. 编码器清零。
3. 陀螺仪有效时 PID 保持 0°；无效时左右轮等速。
4. 第二题按当前要求不启用编码器提前减速，也不使用固定距离硬停车。
5. 至少两路灰度连续 20 ms 检测到黑线后反向制动并结束 A→B。

BC 循迹当前未实现。队友的 BC 循迹模块到达 C 点后调用：

```c
Mission_beginQ2_CD();
```

调用后会启动 C→D，目标航向为 180.0°。该接口声明在：

```c
// mission.h
void Mission_beginQ2_CD(void);
```

注意：

- `Mission_beginQ2_CD()` 只在公共任务处于 `IDLE` 或 `FINISHED` 时生效。
- BC 循迹模块到达 C 点后，应先停止自己的电机输出，再调用此接口。
- DA 循迹未实现。

### 2.4 第三题：A→C

启动按钮：PB23，只按一次。

第三题状态和 PID 独立保存在 `q3_mission.c`，不会复用第一、二题的
PID 积分或阶段变量。

当前 A→C 流程：

1. 按 PB23，发出 A 点声光提示。
2. 先原地校正并稳定在 0°。
3. 原地 PID 转到 38.0°。
4. 进入目标 ±10°后自动降低转弯速度，减少左右摆动。
5. 进入目标 ±3°并连续稳定 80 ms 后开始直行。
6. 保持 38.0°直线 PID。
7. A→C 目标距离为 1269 mm，采用三级速度：
   - 前段：360‰；
   - 剩余约 300 mm：260‰；
   - 剩余约 120 mm：160‰。
8. 满足任一条件后反向制动：
   - 至少两路灰度连续 20 ms 检测到黑线；
   - 编码器距离达到 1269 mm。
9. 在 C 点制动完成后保持停车，等待未来的 CB 循迹模块。

当前没有接入 CB 循迹，因此第三题执行到 C 点后会停在
`Q3_WAIT_CB_TRACE`，不会要求第二次按 PB23。

### 2.5 第三题预留的 B→D 接口

队友的 CB 循迹模块到达 B 点后调用：

```c
Q3Mission_continueFromB();
```

接口声明：

```c
// q3_mission.h
void Q3Mission_continueFromB(void);
```

调用后的预留流程已经写好：

1. 发出 B 点声光提示。
2. 原地校正并稳定在 180.0°。
3. 原地 PID 转到 142.0°。
4. 保持 142.0°直行。
5. 使用与 A→C 相同的对角线距离、三级速度、两路黑线判断和制动。
6. 到 D 点后发出完成声光提示。

对接约束：

- 只有第三题处于 `Q3_WAIT_CB_TRACE` 时调用才生效。
- CB 循迹模块不能再次模拟 PB23 按键。
- 到 B 点后先停止循迹模块自己的电机控制，再调用接口。
- `Q3Mission_continueFromB()` 后，电机控制权交回第三题模块。

### 2.6 第四题

- PB24 已配置为按键并能产生 `KEY_Q4`。
- 当前 `mission.c` 没有处理 `KEY_Q4`，所以按 PB24 不会启动电机。
- 第四题尚未实现。
- 建议第四题复用第三题的“转向 PID + 对角线三级速度 + 黑线/距离停车”
  逻辑，但新增独立的 `q4_mission.c/.h`，不要把四次循环直接塞入
  `mission.c`，也不要修改第一、二题 PID 状态。

## 3. 任务与接口关系

```text
main.c
  └─ 每 1 ms 调用 Mission_task1ms()
       ├─ Keys_task1ms()                  按键扫描
       ├─ Indicator_task1ms()             声光提示
       ├─ Lcd_showYaw()                   LCD 角度
       ├─ 第一、二题公共状态机
       └─ 第三题激活时调用 Q3Mission_task1ms()

BC 循迹模块到达 C
  └─ Mission_beginQ2_CD()

CB 循迹模块到达 B
  └─ Q3Mission_continueFromB()
```

## 5. 引脚定义

### 5.1 电机与 TB6612

| 功能 | MCU 引脚 | 方向/外设 |
|---|---:|---|
| 电机 A IN1 | PB4 | GPIO 输出 |
| 电机 A IN2 | PB5 | GPIO 输出 |
| 电机 B IN1 | PB6 | GPIO 输出 |
| 电机 B IN2 | PB7 | GPIO 输出 |
| TB6612 STBY | PB12 | GPIO 输出，低电平禁用 |
| 左电机 PWM | PA13 | TIMG0 CCP1 |
| 右电机 PWM | PA15 | TIMA1 CCP0 |

`Motor_drive(left, right)` 的输入单位是千分比：

- `360` 表示正转 36%；
- `-280` 表示反转 28%；
- 有效范围会限制在 `-1000~1000`。

### 5.2 编码器

| 功能 | MCU 引脚 | 当前用途 |
|---|---:|---|
| 左轮 E1A | PB25 | 上升沿中断计数 |
| 左轮 E1B | PA23 | GPIO 输入，当前不参与中断解码 |
| 右轮 E2A | PB21 | 上升沿中断计数 |
| 右轮 E2B | PB27 | GPIO 输入，当前不参与中断解码 |

当前不是四路双边沿的真实 AB 正交解码。代码统计 A 相上升沿后乘以
`ENCODER_X4_SCALE = 4`，统一成 1040 计数/圈的等效口径。

因此：

- 可以用于前进距离估算；
- 当前不能判断轮子方向；
- 反向制动产生的脉冲仍会增加计数，但制动阶段不再使用距离；
- 如果以后实现真正的 A/B 双边沿四倍频中断，必须删除软件 `×4`，
  否则距离会再次缩小为四分之一。

距离计算：

```text
平均等效计数 = (左 A 相上升沿 + 右 A 相上升沿) / 2 × 4
距离(mm) = 平均等效计数 × π × 65 / 1040
```

### 5.3 陀螺仪 UART

| 信号 | MCU 引脚 | 接线 |
|---|---:|---|
| UART0 RX | PA11 | 接陀螺仪 TX |
| UART0 TX | PA10 | 接陀螺仪 RX |
| GND | GND | 必须共地 |
| VCC | 建议 5V | 模块典型供电 |

UART 配置：

- 115200 baud；
- 8 数据位；
- 无校验；
- 1 停止位；
- 无流控；
- 只启用 Receive 中断；
- 不启用 DMA done on receive。

数据帧：

```text
5A BB YawL YawH SUM
SUM = 5A + BB + YawL + YawH，取低 8 位
```

角度接口使用 0.1°：

- `Gyro_yawDeciDeg() == 380` 表示 38.0°；
- `1420` 表示 142.0°；
- `1800` 表示 180.0°。

`Gyro_valid()` 只有收到并校验通过至少一个角度帧后才为真。
LCD 目前在无有效帧时也可能显示 0，因此 LCD 显示 0 不等于 UART 一定正常。

### 5.4 灰度仪

| 功能 | MCU 引脚 | 配置 |
|---|---:|---|
| GRAY_CLK | PB8 | GPIO 输出 |
| GRAY_DAT | PB9 | GPIO 输入，上拉 |

- CLK 上升沿让辅助板更新 DAT。
- 低电平代表对应探头看到黑线。
- 至少两路为黑才判定终点黑线。
- 必须连续确认 20 ms，避免瞬时噪声停车。
- 初始化时 CLK 保持低电平超过 1 ms，用于复位辅助板位序。

### 5.5 按键

| 题目 | MCU 引脚 | SysConfig | 有效电平 |
|---|---:|---|---|
| 第一题 | PA18 | 输入、内部上拉 | 按下低 |
| 第二题 | PB22 | 输入、内部上拉 | 按下低 |
| 第三题 | PB23 | 输入、内部上拉 | 按下低 |
| 第四题 | PB24 | 输入、内部上拉 | 按下低 |

按键逻辑：

- 上电后每个按键独立等待松开高电平 100 ms；
- 按下低电平持续 20 ms 产生一次事件；
- 长按只产生一次；
- 松开后才能再次触发。

如果某个按键一直无法使用：

1. 空闲时测该引脚，应接近 3.3V；
2. 按下时应接近 0V；
3. 如果一直为 0V，该按键只会保持未解锁，不会影响其他按键。

### 5.6 LCD、声光与定时器

| 功能 | MCU 引脚 |
|---|---:|
| LCD 背光 BLK | PA16 |
| LCD CLK | PA17 |
| LCD DC | PB15 |
| LCD SDI | PB16 |
| LCD CS | PB17 |
| LCD SDO | PB20 |
| 蓝色 LED | PA7，低电平点亮 |
| 蜂鸣器 | PB26，高电平响 |
| 1 ms 控制定时器 | TIMG12 |

LCD 只局部刷新变化后的角度数字，不再周期性整屏清屏。

## 7. 声光提示约定

- 任务或新点开始：约 100 ms。
- 正常完成：约 300 ms。
- 故障：约 700 ms。

第三题当前会在 A 点启动、C 点停车时提示；未来调用
`Q3Mission_continueFromB()` 时会在 B 点提示，D 点完成时再次提示。

## 8. 队友接入循迹代码的建议

建议新增独立文件，不要把循迹代码写进 `main.c`：

```text
q2_trace_bc.c/.h
q2_trace_da.c/.h
q3_trace_cb.c/.h
q3_trace_da.c/.h
```

推荐接口风格：

```c
void TraceBC_start(void);
Trace_Result TraceBC_task1ms(void);
void TraceBC_stop(void);
```

循迹模块必须非阻塞运行：

- 不允许在函数里长时间 `while` 等待；
- 每次 `task1ms()` 只执行一步；
- 不直接占用主循环；
- 不重复初始化 SysConfig；
- 完成后先停止自身电机输出，再调用直线段衔接接口。

第二题建议衔接：

```text
Q2 A→B 完成
  → 启动 BC 循迹
  → 到 C 点
  → TraceBC_stop()
  → Mission_beginQ2_CD()
```

第三题建议衔接：

```text
Q3 A→C 完成并等待
  → 启动 CB 循迹
  → 到 B 点
  → TraceCB_stop()
  → Q3Mission_continueFromB()
```

## 9. 已知限制与注意事项

1. 当前编码器不是实际的 A/B 四边沿方向解码，而是 A 相上升沿计数后软件乘 4。
2. 左右轮距离取平均值；单侧编码器失效时，距离仍会变化但会明显失真。
3. 当前角度 0°只在首次收到有效陀螺仪帧时捕获，不是模块硬件重新标零。
4. Q1 陀螺仪无效时仍能等速运行；Q3 没有有效陀螺仪会进入故障。
5. Q2 当前无提前减速、无固定距离硬停车，按两路黑线结束。
6. Q3 到 C 点后等待循迹接口期间，公共任务仍认为第三题处于激活状态；
   在接入 CB 循迹前不要期望此时按其他题目按钮切换任务。
7. `Q3_POINT_HOLD_TIMEOUT_MS` 当前已定义但未实际使用，后续可删除或用于转向前超时。
8. `Encoder_averageDistanceMm()` 使用整数 π≈3.1416，满足当前毫米级分段控制。
9. 修改轮径、编码器计数口径后，必须重新验证所有毫米距离和减速点。
10. 不要在 `releases` 中解压源码到工程根目录，否则 CCS 可能把重复 `.c` 文件加入构建。

## 11. 对接时建议的责任边界

你当前负责：

- 第一题 A→B；
- 第二题 A→B、C→D；
- 第三题 A→C；
- 第三题未来 B→D 直线与转向；
- 编码器距离、陀螺仪直线 PID、终点制动。

队友循迹模块负责：

- 第二题 BC、DA；
- 第三题 CB、DA；
- 到达交接点后停止自己的电机输出；
- 调用本文指定的衔接接口。

整车集成时必须明确同一时刻只有一个模块拥有电机控制权，不能让循迹模块和
`Mission_task1ms()` 同时调用 `Motor_drive()`。
